API Fundamentals

What is an API?

API is an acronym for Application Programming Interface. An API is a tool that allows applications to communicate with each other. The API operating principle is very simple. An application sends data (known as a request) to a server. The server processes this information and sends back a response. The application then interprets that data and presents it in a readable way.

The response usually comes in JSON format. JSON is an open standard data format that is lightweight and human-readable.

What is REST?

REST (Representational State Transfer) is a set of web API architecture principles, or constraints, that dictate how web service applications communicate over the internet. For this purpose, REST API uses the HTTP protocol between client and host server at a predefined set of URLs.

Request Structure

  • Method: HTTP verbs (GET, PUT, POST, etc.)
  • URL: resource and parameters
  • Headers: meta-data of the request
  • Body: content of the request

Response Structure

  • Status Code: HTTP response code
  • Headers: meta-data of the response
  • Body: content of the response

HTTP Verbs

Verb Description
GET Used to read (retrieve) data on the server.
POST Used to create data.
PUT Used to update data.
DELETE Used to delete data.

URL Structure

All Bündnis API requests follow a predictable pattern:

https://api.buendnis.de /v1 /claims ?status=open

Response Codes

HTTP status codes are grouped into classes (1xx to 5xx). The most common responses are:

Code Status Description
200 OK The request has succeeded
201 Created The request has been fulfilled and resulted in a new resource being created
400 Bad Request The request could not be understood due to incorrect syntax
401 Unauthorized The request requires user authentication
403 Forbidden The client does not have access rights to the content
404 Not Found The server has not found anything matching the Request-URI
429 Too Many Requests The user has sent too many requests in a given amount of time ("rate limiting")
500 Internal Server Error The server encountered an unexpected condition
503 Service Unavailable The server is currently unable to handle the request

Glossary

Authentication The verification process to confirm the identity and permissions of a user.
Endpoint A specific URL within an API that represents a resource or a specific functionality.
Rate Limiting A technique to control the number of API requests a user can make within a specified timeframe.

User Guide

Navigating the API catalog and documentation.

View of unauthorized user

  • Step 1: Start on the Homepage.
  • Step 2: Click on API Catalog.
  • Step 3: The API catalog is only visible for authorized users.

Sign in and sign up

  • Step 4: Click on log in.
  • Step 5: Here you can see the log in mask. Enter credentials or sign up.
  • Step 6: If you do not have credentials yet, click on Create an account.
  • Step 7: Type in your sign up credentials and click on Sign up.
  • Step 8: Set up your authenticator app for MFA. Your registration will be activated after a review from the admin.
  • Step 9: Type in your log in credentials and click sign in.
  • Step 10: Enter the code from your authenticator app.

View of the authorized user

  • Step 11: You will be redirected to the homepage. Click on API catalog to see all APIs that belong to your settings.
  • Step 12: You can click on Demo for example and filter your APIs.
  • Step 14: Click on API DOCUMENTATION.
  • Step 15: Here, you can find the API description expanded and see the full specification of the current version.

Access to documentation and further functions

  • Step 17: Click on Documentation to build a common consensus of terms.
  • Step 21: Click on FAQs to find frequently asked questions.
  • Step 23: For any individual purpose don't hesitate to contact us.

Usage is currently limited to contracted business partners. You must have a valid contractual agreement with Bündnis to register for an account and access the specifications.
