Introduction to Bündnis API

Our APIs are built with enterprise-grade security and developer efficiency in mind. Follow the structured guides below to integrate our services into your insurance workflow.

Notice: Production access requires a signed Framework Agreement. Use the Sandbox for development and testing.

Core Principles

  • RESTful Architecture: Standard HTTP methods and status codes.
  • JSON Format: Simplified data exchange for all endpoints.
  • Security First: mTLS and OAuth 2.1 by default.

Authentication & Security

We use OAuth 2.1 for authorization. To access our APIs, you must first obtain a client token.

Token Request Example
curl -X POST https://api.buendnis.de/v1/oauth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=client_credentials&client_id=YOUR_ID&client_secret=YOUR_SECRET"

API Protocols & Versioning

All APIs are versioned to ensure backward compatibility. Use the version prefix in the URL (e.g., /v1/).

Method Endpoint Description
GET /claims List all insurance claims
POST /policy/create Create a new insurance policy

Best Practices

To optimize your integration, please follow these industry-standard best practices.

  • Implement exponential backoff for rate-limited requests.
  • Cache frequently accessed metadata like policy types.
  • Use the test environment for all initial development.
